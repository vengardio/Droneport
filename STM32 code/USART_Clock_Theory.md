# Настройка тактирования USART на STM32F401

## Архитектура шин

```
         SYSCLK (42 МГц)
            │
       ┌────┴────┐
       │   AHB   │  ← делитель HPRE (1, 2, 4, ... 512)
       │  42 МГц │
       └────┬────┘
            │
    ┌───────┴───────┐
    │               │
┌───┴───┐     ┌─────┴─────┐
│ APB1  │     │   APB2    │
│ Low   │     │   High    │
│speed  │     │  speed    │
│42 МГц │     │  42 МГц  │
└───────┘     └───────────┘
```

- **APB1** (low-speed, макс 42 МГц) — TIM2-5, USART2, I2C, SPI2/3
- **APB2** (high-speed, макс 84 МГц) — TIM1/9/10/11, **USART1/6**, SPI1/4, ADC

**USART1 висит на шине APB2.**

Делители настраиваются в регистре `RCC->CFGR`:
- `HPRE` [7:4] — делитель AHB (от SYSCLK)
- `PPRE1` [12:10] — делитель APB1 (от AHB)
- `PPRE2` [15:13] — делитель APB2 (от AHB)

## Текущая конфигурация клоков (rcc.c)

```
HSE = 4 МГц (внешний кварц)
  │
  ├─ PLLM = 2   →  4 / 2 = 2 МГц      (вход PLL, норма: 1-2 МГц)
  ├─ PLLN = 42  →  2 × 42 = 84 МГц    (VCO)
  ├─ PLLP = 2   →  84 / 2 = 42 МГц    (SYSCLK)
  └─ PLLQ = 2   →  84 / 2 = 42 МГц    (USB/SDIO, не используется)

HPRE  = 0 → AHB  = SYSCLK / 1 = 42 МГц
PPRE1 = 0 → APB1 = AHB / 1    = 42 МГц
PPRE2 = 0 → APB2 = AHB / 1    = 42 МГц
```

Все делители стоят в 0 — деление на 1. Частота на всех шинах = **42 МГц**.

## Как считается BRR (делитель бодрейта USART)

Регистр `BRR` определяет, на сколько делить тактовую частоту шины чтобы получить нужный бодрейт.

### Формула из Reference Manual

```
USARTDIV = f_CK / (8 × (2 - OVER8) × baudrate)
```

Где:
- `f_CK` — частота шины, к которой подключён USART
- `OVER8` — бит оверсэмплинга в `CR1`. По умолчанию 0 (oversampling ×16)

При OVER8 = 0 (дефолт) формула упрощается:

```
USARTDIV = f_CK / (16 × baudrate)
```

### Структура регистра BRR

BRR — это fixed-point число:

```
BRR [15:4]  = мантисса (целая часть USARTDIV)
BRR [3:0]   = дробная часть (дробь USARTDIV × 16)
```

**Совет:** если записать `USARTDIV × 16` целым числом в BRR, STM32 сам разберёт мантиссу и дробь, потому что 4 нижних бита — это и есть дробная часть, умноженная на 16.

Отсюда итоговая формула для записи в BRR:

```c
BRR = (16 × f_CK) / baudrate
```

### Текущий код (usart1.c)

```c
uint32_t usart1div = (16 * 42000000) / boudrate;
USART1->BRR = usart1div;
```

Формула **правильная** для f(APB2) = 42 МГц и OVER8 = 0.

### Примеры расчёта

| Baudrate | BRR = (16 × 42M) / baud | Hex    | Реальный бод     | Ошибка  |
|----------|--------------------------|--------|------------------|---------|
| 420000   | 1600                     | 0x0640 | 420000           | 0%      |
| 115200   | 5833                     | 0x16C9 | 115218           | 0.016%  |
| 9600     | 70000                    | 0x11170| 9600             | 0%      |

Допустимая ошибка бодрейта — до ~2-3%. Всё что меньше 1% — идеально.

## Нюанс: таймерный множитель ×2

На STM32F4 есть важное правило для таймеров:

> Если делитель APBx **не равен 1**, то тактовая частота таймеров на этой шине **удваивается**.

```
APBx prescaler = 1  → TIM clock = APBx
APBx prescaler > 1  → TIM clock = APBx × 2
```

### Текущая проблема

Сейчас APB1 prescaler = 1 (деление на 1), значит:
- TIM2 clock = APB1 = 42 МГц (множитель ×2 **не** применяется)
- PSC = 83 → тик = 42000000 / (83+1) = **500 кГц = 2 мкс/тик**

Комментарий в коде ("TIM clock = 84 МГц") **не соответствует** реальности при текущих настройках.

### Как исправить

Два варианта:

**Вариант А** — поменять PSC под реальную частоту:
```c
TIM2->PSC = 41;  // 42 МГц / (41+1) = 1 МГц = 1 мкс/тик
```

**Вариант Б** — разогнать SYSCLK до 84 МГц и поделить APB1:
```c
// В ClockInit():
// PLLN = 84 вместо 42 → SYSCLK = 84 МГц
// PPRE1 = 0b100 (деление на 2) → APB1 = 42 МГц
// Тогда TIM clock = 42 × 2 = 84 МГц, и PSC = 83 будет верным
```

## Общий алгоритм настройки USART

1. Включить тактирование USART в соответствующем регистре `RCC->APBxENR`
2. Выключить USART (`CR1 &= ~UE`) перед настройкой
3. Настроить формат кадра: длина слова (`M`), чётность (`PCE`), стоп-биты (`STOP`)
4. Рассчитать и записать BRR: `BRR = (16 × f_APBx) / baudrate`
5. Включить приёмник/передатчик (`RE`, `TE`), прерывания если нужны (`RXNEIE`)
6. Включить USART (`CR1 |= UE`)
7. Включить прерывание в NVIC если используется
